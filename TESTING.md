# Testing

This document describes how to verify the skills in this repo produce useful output before relying on them against real Salesforce data.

## Quick test (5 minutes, no setup)

If you just want to see what the skills produce, read the sample outputs:

- [salesforce-revops-audit sample](skills/salesforce-revops-audit/examples/sample_output.md)
- [forecast-call-prep sample](skills/forecast-call-prep/examples/sample_output.md)
- [pipeline-hygiene-audit sample](skills/pipeline-hygiene-audit/examples/sample_output.md)
- [deal-investigator sample](skills/deal-investigator/examples/sample_output.md)

These were generated by running deterministic Python logic that mirrors what each SKILL.md instructs Claude to do, against the bundled [Northwind Cloud](data/) synthetic dataset. They represent what each skill should produce.

## Full test (30 minutes, requires Claude Code or Claude Desktop)

### Prerequisites

- [Claude Code](https://docs.claude.com/en/docs/claude-code) installed and authenticated
- An active Claude Pro or Max subscription
- This repo cloned to a known location on disk

### Test sequence

Navigate to the repo and start a Claude Code session:

```bash
cd revops-skills
claude
```

Run the four tests in order. After each, compare Claude's output to the corresponding sample in `skills/<skill-name>/examples/sample_output.md`.

#### Test 1: salesforce-revops-audit

```
Read skills/salesforce-revops-audit/SKILL.md and follow its instructions
exactly. Run the audit against the synthetic data in the data/ directory.
Today's date is May 20, 2026. Produce the full audit report.
```

Pass criteria: overall grade B or C, recommends `pipeline-hygiene-audit` first in the remediation queue, identifies pipeline concentration in a single rep.

#### Test 2: forecast-call-prep

```
Read skills/forecast-call-prep/SKILL.md and follow its instructions
exactly. Use the synthetic data in data/. Today's date is May 20, 2026.
Produce the weekly forecast call memo.
```

Pass criteria: optimist reps (Diego, Tomas, Hannes) appear in the Risks section. Sandbagger reps (Priya, Lakshmi) appear in the Upsides section. Recommendation takes a position rather than hedging.

#### Test 3: pipeline-hygiene-audit

```
Read skills/pipeline-hygiene-audit/SKILL.md and follow its instructions
exactly. Use the synthetic data in data/. Today's date is May 20, 2026.
Produce the per-rep hygiene punchlist.
```

Pass criteria: at least 30 high-severity items flagged with opp IDs, top three reps ranked by issue weight, specific remediation commitment for the manager.

#### Test 4: deal-investigator

```
Read skills/deal-investigator/SKILL.md and follow its instructions exactly.
Target opportunity is opp_0029. Use the synthetic data in data/. Today's
date is May 20, 2026. Produce the deal review memo.
```

Pass criteria: all five questions answered, recommendation is Pipeline or Omit (not Commit), risk score is 3 or higher.

#### Test 5: chained flow (optional)

Start a fresh session and prompt:

```
I just took over RevOps at Northwind Cloud. I need to assess the state
of our Salesforce org and figure out where to focus first. The data is
in data/. Today is May 20, 2026.
```

Pass criteria: Claude proposes running `salesforce-revops-audit` first rather than jumping to a specific remediation skill.

## Test against your own Salesforce org

Once you have validated the skills work against the synthetic dataset, you can run them against real Salesforce data via the [official Salesforce-hosted MCP server](https://developer.salesforce.com/blogs/2026/04/salesforce-hosted-mcp-servers-are-now-generally-available) or the [Salesforce DX MCP](https://developer.salesforce.com/blogs/2025/06/introducing-mcp-support-across-salesforce).

Each SKILL.md includes an Appendix A with the SOQL queries the skill expects. Configure your Salesforce MCP server in Claude Code or Claude Desktop, then invoke the skill normally. The skill will detect the MCP and use it instead of the local sample data.

This validation is not part of the v0.1 test suite. The Salesforce MCP integration is designed-for but not yet validated against a live org. Verification against a [Salesforce Developer Edition](https://developer.salesforce.com/signup) org is planned for v0.2.

## Reporting test failures

If a skill produces output that materially differs from the sample, open an issue with:

1. Which skill failed
2. The exact prompt used
3. The full output Claude produced
4. What specifically was wrong (referenced against the pass criteria above)

Most failures are SKILL.md instruction clarity issues, not logic bugs. They are usually one revision away from passing.
